# Aurora 重构版代码规范

> 本文档适用于 `重构版/` 目录下的 Aurora 博客项目，后端目标技术栈为 **JDK 25 + Spring Boot 3.x**，前台为 **Vue 3 + TypeScript**，后台为 **Vue 2**（后续计划迁移至 Vue 3）。

---

## 一、通用规范

### 1.1 编码与换行

- 文件编码：**UTF-8**
- 换行符：**LF**（`\n`），禁止 CRLF
- 缩进：**4 个空格**（后端 Java），**2 个空格**（前端 Vue/TS/JS/CSS）
- 行尾不允许有多余空格
- 文件末尾保留一个空行

### 1.2 文件命名

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| Java 类 | 大驼峰 | `ArticleServiceImpl.java` |
| Java 接口 | 大驼峰，通常以 `Service`/`Mapper` 等结尾 | `ArticleService.java` |
| Vue 组件 | 大驼峰 | `ArticleCard.vue` |
| TypeScript/JavaScript 文件 | 小驼峰 | `articleStore.ts`、`api.ts` |
| CSS/SCSS 文件 | 小写，中划线连接 | `article-card.scss` |
| 常量类 | 全大写，下划线连接 | `RedisConstant.java` |

### 1.3 目录结构

后端目录保持与原始项目一致：

```
com.aurora
├── annotation      # 自定义注解
├── aspect          # AOP
├── config          # 配置类
├── constant        # 常量
├── consumer        # MQ 消费者
├── controller      # 控制器
├── entity          # 数据库实体
├── enums           # 枚举
├── event           # 事件
├── exception       # 自定义异常
├── filter          # 过滤器
├── handler         # 处理器
├── interceptor     # 拦截器
├── mapper          # MyBatis-Plus Mapper
├── model           # DTO / VO
├── quartz          # 定时任务
├── service         # 服务接口与实现
├── strategy        # 策略模式接口与实现
├── util            # 工具类
└── AuroraSpringbootApplication.java
```

## 二、后端 Java 规范

### 2.1 基础规范

- 使用 **JDK 25** 特性，但避免使用过于实验性的语法。
- 包名统一小写：`com.aurora.*`
- 类名、接口名使用大驼峰。
- 方法名、变量名使用小驼峰。
- 常量名使用全大写下划线连接。
- 禁止拼音命名、无意义缩写（如 `a1`、`xx`、`tmp` 等）。

### 2.2 类与接口

- 类长度不超过 **500 行**，超过需拆分。
- 方法长度不超过 **80 行**，超过需抽取私有方法。
- 单个方法参数不超过 **5 个**，超过使用 DTO 封装。
- Service 接口与实现分离，实现类以 `Impl` 结尾。

```java
// 推荐
public interface ArticleService {
    PageResultDTO<ArticleCardDTO> listArticles(ConditionVO condition);
}

@Service
public class ArticleServiceImpl implements ArticleService {
    // ...
}
```

### 2.3 代码格式

- 左大括号不换行：

```java
public class UserService {
    public void save(User user) {
        if (user == null) {
            throw new BizException("用户不能为空");
        }
    }
}
```

- 单行 `if/for/while` 也必须加花括号：

```java
// 推荐
if (user == null) {
    return;
}

// 禁止
if (user == null) return;
```

### 2.4 异常处理

- 禁止直接吞掉异常：

```java
// 禁止
try {
    doSomething();
} catch (Exception e) {
    // 什么都不做
}
```

- 必须记录日志或抛出业务异常：

```java
// 推荐
try {
    doSomething();
} catch (IOException e) {
    log.error("操作失败", e);
    throw new BizException("操作失败");
}
```

- 优先使用自定义业务异常 `BizException`，避免直接抛出 `RuntimeException`。
- 减少 `@SneakyThrows` 使用，仅允许在纯工具类中使用。

### 2.5 日志规范

- 统一使用 SLF4J + Logback：

```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class UserService {
    private static final Logger log = LoggerFactory.getLogger(UserService.class);
}
```

- 或者使用 Lombok `@Slf4j`：

```java
import lombok.extern.slf4j.Slf4j;

@Slf4j
@Service
public class UserService {
    public void doSomething() {
        log.info("开始执行...");
    }
}
```

- 禁止使用 `System.out.println`、`printStackTrace()`。
- 日志占位符用法：

```java
// 推荐
log.info("用户登录成功，用户ID: {}", userId);

// 禁止
log.info("用户登录成功，用户ID: " + userId);
```

### 2.6 返回值与 Controller

- Controller 统一返回 `ResultVO<T>`。
- 禁止在 Controller 中写业务逻辑，只负责参数校验和调用 Service。
- 使用 `@Valid` / `@Validated` 进行参数校验。

```java
@RestController
@RequestMapping("/api/articles")
public class ArticleController {

    @Autowired
    private ArticleService articleService;

    @GetMapping("/all")
    public ResultVO<PageResultDTO<ArticleCardDTO>> listArticles(@Valid ConditionVO condition) {
        return ResultVO.ok(articleService.listArticles(condition));
    }
}
```

### 2.7 数据库与 MyBatis-Plus

- 实体类使用 `@TableName` 和 `@TableId` 注解。
- 主键统一使用 `Integer` 或 `Long`，根据数据量选择。
- 时间字段使用 `LocalDateTime`。
- 逻辑删除字段统一命名 `is_delete`。
- Mapper 接口不编写 XML，优先使用 MyBatis-Plus 提供的 API 和注解。
- 复杂 SQL 使用 `@Select` 注解或 XML，XML 放在 `resources/mapper/` 下。

```java
@Data
@TableName("t_article")
public class Article {
    @TableId(type = IdType.AUTO)
    private Integer id;
    private String articleTitle;
    private String articleContent;
    private Integer isDelete;
    private LocalDateTime createTime;
    private LocalDateTime updateTime;
}
```

### 2.8 安全配置

- Spring Security 配置使用 `SecurityFilterChain` Bean，禁止继承 `WebSecurityConfigurerAdapter`。
- JWT Secret 必须使用高强度随机字符串，禁止硬编码简单字符串。
- 密码必须使用 `BCryptPasswordEncoder` 加密存储。

### 2.9 依赖与 import

- 禁止 `import *`，必须显式导入。
- 删除未使用的 import。
- 优先使用 Spring 官方 Starter，避免引入重复功能的第三方库。

### 2.10 Lombok 使用

- 允许使用 `@Data`、`@Getter`、`@Setter`、`@NoArgsConstructor`、`@AllArgsConstructor`、`@Builder`、`@Slf4j`。
- 实体类建议用 `@Data` 或 `@Getter/@Setter`。
- 不建议在 DTO/VO 中使用 `@Data` 时同时暴露所有 setter，必要时使用 `@Getter` + 特定 `@Setter`。

## 三、前端规范

### 3.1 通用规范

- 使用 **Vue 3 + TypeScript**（前台），后台暂用 Vue 2，后续统一迁移。
- 字符串优先使用单引号。
- 语句结束必须加分号。
- 缩进 2 个空格。

### 3.2 Vue 单文件组件

- 标签顺序：`template` → `script` → `style`。
- `script` 使用 `setup` 语法糖（Vue 3）：

```vue
<template>
  <div class="article-card">
    <h2>{{ title }}</h2>
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue'

const title = ref('文章标题')
</script>

<style scoped>
.article-card {
  padding: 16px;
}
</style>
```

### 3.3 TypeScript 类型

- 禁止滥用 `any`，必须定义明确类型。
- API 请求参数和返回数据必须定义接口：

```typescript
// types/article.ts
export interface Article {
  id: number
  articleTitle: string
  articleContent: string
  createTime: string
}

export interface ArticleQueryParams {
  page: number
  size: number
  categoryId?: number
}
```

- Store 必须定义 State 类型：

```typescript
import { defineStore } from 'pinia'
import type { Article } from '@/types/article'

interface ArticleState {
  articles: Article[]
  loading: boolean
}

export const useArticleStore = defineStore('article', {
  state: (): ArticleState => ({
    articles: [],
    loading: false
  }),
  actions: {
    async fetchArticles() {
      // ...
    }
  }
})
```

### 3.4 API 封装

- 统一封装 Axios 实例，禁止直接调用 `axios.get/post`：

```typescript
// utils/request.ts
import axios from 'axios'

const request = axios.create({
  baseURL: '/api',
  timeout: 10000
})

request.interceptors.request.use(/* ... */)
request.interceptors.response.use(/* ... */)

export default request
```

```typescript
// api/article.ts
import request from '@/utils/request'
import type { Article, ArticleQueryParams } from '@/types/article'
import type { Result } from '@/types/result'

export const getArticles = (params: ArticleQueryParams) => {
  return request.get<Result<Article[]>>('/articles/all', { params })
}
```

### 3.5 组件规范

- 组件名使用大驼峰，与文件名一致。
- Props 必须定义类型和默认值（Vue 3 使用 `withDefaults` 或 `defineProps`）。
- 组件职责单一，避免一个组件超过 400 行。

```vue
<script setup lang="ts">
interface Props {
  title: string
  visible?: boolean
}

const props = withDefaults(defineProps<Props>(), {
  visible: false
})
</script>
```

### 3.6 样式规范

- 使用 SCSS 或 Tailwind CSS。
- 类名使用 BEM 命名法或 Tailwind 工具类。
- 组件样式必须加 `scoped`，避免污染全局。

```vue
<style scoped lang="scss">
.article-card {
  &__title {
    font-size: 18px;
  }
}
</style>
```

## 四、Git 提交规范

### 4.1 分支模型

- `main`：主分支，保持可发布状态
- `develop`：开发分支
- `feature/*`：功能分支
- `fix/*`：Bug 修复分支
- `docs/*`：文档修改

### 4.2 Commit Message 格式

采用 **Conventional Commits**：

```
<type>(<scope>): <subject>

<body>

<footer>
```

类型（type）：

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | 修复 Bug |
| `docs` | 文档变更 |
| `style` | 代码格式调整（不影响功能） |
| `refactor` | 重构 |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建/工具/依赖调整 |
| `security` | 安全相关 |

示例：

```
feat(backend): 升级 Spring Boot 到 3.4.0

- 更新 pom.xml 中的 Spring Boot 版本
- 迁移 javax 到 jakarta 命名空间
- 重写 WebSecurityConfig 为 SecurityFilterChain

Closes #1
```

### 4.3 提交粒度

- 每次提交只做一件事。
- 禁止将不相关的修改放在同一个提交中。
- 重构、功能、文档修改应分开提交。

## 五、API 设计规范

### 5.1 URL 设计

- 使用 RESTful 风格，全部小写，中划线连接。
- 资源名词用复数形式。

```
GET    /api/articles          # 查询文章列表
GET    /api/articles/{id}     # 查询单个文章
POST   /api/articles          # 创建文章
PUT    /api/articles/{id}     # 更新文章
DELETE /api/articles/{id}     # 删除文章
```

### 5.2 状态码

| HTTP 状态码 | 含义 |
|-------------|------|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 5.3 返回结构

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

错误返回：

```json
{
  "code": 500,
  "message": "文章不存在",
  "data": null
}
```

## 六、安全规范

### 6.1 后端安全

- 所有用户输入必须校验。
- 防止 SQL 注入：使用 MyBatis-Plus API 或参数化查询，禁止字符串拼接 SQL。
- 防止 XSS：输出到前端的内容做转义处理。
- 防止 CSRF：使用 JWT + 无状态会话，或开启 CSRF 防护。
- 敏感配置（密码、密钥、Token）禁止提交到 Git。
- JWT Secret 长度至少 256 位。

### 6.2 前端安全

- 用户输入内容渲染前做 XSS 过滤。
- 避免在代码中硬编码密钥、APP ID 等敏感信息。
- Token 存储在 `sessionStorage` 或内存中，避免使用 `localStorage` 长期保存。

## 七、数据库规范

### 7.1 命名

- 表名使用 `t_` 前缀，小写下划线连接：`t_article`、`t_user_auth`
- 字段名小写下划线连接：`article_title`、`create_time`
- 主键统一命名：`id`
- 逻辑删除字段：`is_delete`
- 时间字段：`create_time`、`update_time`

### 7.2 字段类型

| 含义 | 推荐类型 |
|------|----------|
| 主键 | INT / BIGINT |
| 布尔值 | TINYINT(1) |
| 大文本 | TEXT / LONGTEXT |
| 时间 | DATETIME |
| IP 地址 | VARCHAR(50) |
| URL | VARCHAR(500) |

### 7.3 索引

- 主键必须有索引。
- 外键字段、查询条件字段、排序字段建议加索引。
- 索引命名：`idx_表名_字段名`

## 八、测试规范

### 8.1 后端测试

- 核心 Service 必须编写单元测试。
- Controller 必须编写集成测试。
- 测试类命名：`XxxServiceTest`、`XxxControllerTest`。
- 使用 JUnit 5 + Mockito。

```java
@SpringBootTest
class ArticleServiceTest {

    @Autowired
    private ArticleService articleService;

    @Test
    void shouldReturnArticlesWhenListArticles() {
        // given
        ConditionVO condition = new ConditionVO();
        condition.setPage(1);
        condition.setSize(10);

        // when
        PageResultDTO<ArticleCardDTO> result = articleService.listArticles(condition);

        // then
        assertNotNull(result);
    }
}
```

### 8.2 前端测试

- 推荐使用 Vitest 进行单元测试。
- 工具函数必须有测试覆盖。
- 复杂组件建议编写组件测试。

## 九、文档规范

- 新增功能必须同步更新文档。
- 复杂业务逻辑需要补充注释说明 **为什么**（Why），而不是 **做什么**（What）。
- 接口变更必须更新 API 文档（Knife4j / SpringDoc）。
- 部署配置变更必须更新部署文档。

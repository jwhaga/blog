# Aurora 重构版 — 项目启动与常见问题

## 一、整体架构

浏览器访问 localhost:8080（博客） / localhost:8083（管理后台）
  -> proxy: /api/* 转发到后端
  -> 后端 java -jar port 8081
  -> Docker 基础设施 (MySQL / Redis / RabbitMQ / ES)

## 二、启动顺序

### 2.1 启动 Docker
docker compose up -d

D:\Develop\JDK25\jdk-25.0.3+9 = "D:\Develop\JDK25\jdk-25.0.3+9"

## 四、常见问题

### 4.1 浏览器黑屏
原因：后端未启动时，前端请求 /api 获取网站配置失败，若代码用 !.（非空断言）会崩溃。
当前状态：已将 websiteConfig!. 全部替换为 websiteConfig?.，null 时不崩溃，但部分区域空白。
根治：启动后端。

### 4.2 Cannot read properties of null (reading 'websiteTitle')
原因：演示站 www.linhaojun.top 已不可用，或后端未启动。
解决：启动本地后端，proxy 已指向 localhost:8081。

### 4.3 端口被占用
netstat -ano | findstr :8080
taskkill /PID 对应PID /F

### 4.4 npm install 失败
Remove-Item -Recurse -Force node_modules
Remove-Item -Force package-lock.json
npm install

### 4.5 Docker 无法启动
docker logs 容器名  # 查看具体日志
docker compose down && docker compose up -d  # 重建

## 五、代理配置

博客和后台的 vue.config.js 中 proxy 已配置为 localhost:8081（本地后端）。
原演示站 www.linhaojun.top 已不可用。

## 六、常用命令

启动 Docker：   docker compose up -d
停止 Docker：   docker compose stop
编译后端：      mvn clean compile -f aurora-springboot\pom.xml
测试后端：      mvn clean test -f aurora-springboot\pom.xml
打包后端：      mvn clean package -DskipTests -f aurora-springboot\pom.xml
启动后端：      java -jar aurora-springboot\target\aurora-springboot-0.0.1.jar --spring.profiles.active=dev --server.port=8081
启动博客：      cd aurora-vue/aurora-blog && npm run serve
启动后台：      cd aurora-vue/aurora-admin && npm run serve

# Aurora 关键数据流文档

> 本文档描述系统中最重要的几条数据流链路。

---

## 一、文章搜索同步流（MySQL binlog → Maxwell → RabbitMQ → ES）

### 1.1 链路总览

```
MySQL binlog
    ↓ Maxwell 监听 binlog
    ↓ 输出 JSON 格式变更事件
    ↓ RabbitMQ（maxwell Exchange → Queue）
    ↓ MaxWellConsumer.@RabbitHandler
    ↓ ElasticsearchMapper.save() / deleteById()
    ↓ Elasticsearch 索引更新
```

### 1.2 消费者逻辑

```
MaxWellConsumer.process(byte[] data):
    1. 反序列化为 MaxwellDataDTO
    2. 取出 data 字段，转为 Article 实体
    3. 判断 type:
       - insert / update → 写入 ES
       - delete → 从 ES 删除
```

### 1.3 重构注意事项

- ES 7.9.2 → 8.15.x：检查 ES Mapper API 兼容性
- ES 8.x 默认开启安全认证，本地开发需关闭
- 9200 端口被 Cpolar 占用，ES 需改端口
- Maxwell 版本需与 ES 8.x 和 Spring Boot 3.x 兼容

## 二、用户登录认证流（JWT + Redis + Spring Security）

### 2.1 登录流程

```
1. 用户提交用户名+密码（或 QQ code）
2. UserAuthService 校验凭证
   a) 邮箱密码：BCrypt 校验
   b) QQ 登录：code → access_token → openID
3. 验证通过后构建 UserDetailsDTO
4. TokenService.createToken() 生成 JWT（HS256）
5. TokenService.refreshToken() 存入 Redis
6. 返回 JWT + 用户信息
```

### 2.2 请求鉴权流程

```
1. 请求头携带 Authorization: Bearer <token>
2. JwtAuthenticationTokenFilter 拦截
   a) 提取并解析 token
   b) 从 Redis 查询用户信息
   c) 构建 Authentication 放入 SecurityContext
3. SecurityFilterChain 鉴权
   - 公开接口免认证
   - 需要认证的接口校验
   - 动态权限校验（资源-角色）
4. TokenService.renewToken() 自动续期
```

### 2.3 重构注意事项

- jjwt 0.9.0 → 0.12.x：signWith 和 parser API 变更
- WebSecurityConfigurerAdapter 废弃 → SecurityFilterChain Bean
- authorizeRequests() → authorizeHttpRequests()
- antMatchers() → requestMatchers()
- javax.servlet → jakarta.servlet 命名空间迁移
- JWT Secret 建议使用 256 位以上随机密钥

## 三、评论通知流（MQ 异步邮件）

```
1. 访客提交评论 → POST /comments/save
2. CommentService.saveComment() 保存评论到数据库
3. 发送 RabbitMQ 消息到通知队列
4. CommentNoticeConsumer 异步消费消息
5. 通过 JavaMailSender 发送通知邮件
```

---

## 四、访客统计流

```
1. 用户访问前台页面 → POST /report
2. AuroraInfoService.report()：
   - Redis HyperLogLog 统计 UV（去重）
   - Redis 自增 PV
   - 更新 t_unique_view 表（每日一条）
3. Quartz 定时任务每天凌晨汇总前一日数据
```

### Redis 数据结构使用总览

| 结构 | 用途 | 示例 Key 模式 |
|-----|------|--------------|
| HyperLogLog | UV 统计 | unique_visitor:2026-07-07 |
| String | PV 计数 | page_view |
| ZSet | 文章排行榜 | article_rank |
| Hash | 用户会话 | login_user (field=userId) |
| BitMap | 签到 | user_sign:202607 |

## 五、文件上传流（策略模式）

```
1. Controller 接收 MultipartFile
2. UploadStrategyContext.executeUploadStrategy()
3. 根据 upload.mode 选择实现策略
4. 上传到 OSS 或 MinIO，返回 URL
```

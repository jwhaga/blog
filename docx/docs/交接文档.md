### WebSecurityConfig 清理（authorizeRequests → authorizeHttpRequests）
- 关键文件：[WebSecurityConfig.java](file:///d:/0trea-make-app/小学期作业/重构版/aurora-springboot/src/main/java/com/aurora/config/WebSecurityConfig.java)、[DynamicAuthorizationManagerImpl.java](file:///d:/0trea-make-app/小学期作业/重构版/aurora-springboot/src/main/java/com/aurora/handler/DynamicAuthorizationManagerImpl.java)、[FilterInvocationSecurityMetadataSourceImpl.java](file:///d:/0trea-make-app/小学期作业/重构版/aurora-springboot/src/main/java/com/aurora/handler/FilterInvocationSecurityMetadataSourceImpl.java)、[AccessDecisionManagerImpl.java](file:///d:/0trea-make-app/小学期作业/重构版/aurora-springboot/src/main/java/com/aurora/handler/AccessDecisionManagerImpl.java)
- **背景**：原 `WebSecurityConfig` 使用 `authorizeRequests().withObjectPostProcessor(FilterSecurityInterceptor.class, ...)` 注入自定义 `AccessDecisionManager` 和 `FilterInvocationSecurityMetadataSource`，实现动态权限校验。`authorizeHttpRequests` 内部用 `AuthorizationFilter`，不再使用 `FilterSecurityInterceptor`，因此 `withObjectPostProcessor(FilterSecurityInterceptor.class, ...)` 在新 API 下不生效。
- **方案**：新建 `DynamicAuthorizationManagerImpl` 实现 `AuthorizationManager<RequestAuthorizationContext>`，委托 `FilterInvocationSecurityMetadataSourceImpl` 加载 URL→角色映射、委托 `AccessDecisionManagerImpl` 做决策。
- **Bug 修复**：Spring Security 6.x 在 `FilterInvocation` 构造函数加了 `Assert.isTrue` 检查，不允许 response/chain 为 null。因此 `FilterInvocationSecurityMetadataSourceImpl` 提取出 `getAttributes(HttpServletRequest)` 方法，`DynamicAuthorizationManagerImpl` 直接传 request，不再构造 `FilterInvocation`。
- **行为等价性**：
  - URL 无匹配规则（如 `/users/login`、`/doc.html`）：`getAttributes` 返回 `null` → `AuthorizationDecision(true)` 放行。
  - URL 有匹配规则：委托 `AccessDecisionManagerImpl.decide()`，命中即放行；未命中抛 `AccessDeniedException("权限不足")`，由 `ExceptionTranslationFilter` 交给 `accessDeniedHandler` / `authenticationEntryPoint`。
  - 缓存刷新：`ResourceServiceImpl` / `RoleServiceImpl` 仍通过 `FilterInvocationSecurityMetadataSourceImpl.clearDataSource()` 刷新，未受影响。
- 移除了 `WebSecurityConfig` 中两个迁移后变为死代码的 `@Bean`：`securityMetadataSource()` 和 `accessDecisionManager()`。`AccessDecisionManagerImpl` 和 `FilterInvocationSecurityMetadataSourceImpl` 两个类保留不动（仍是 `@Component`，由 `DynamicAuthorizationManagerImpl` 注入使用）。

  | 修复下游类型不匹配 | ExceptionLogServiceImpl、FriendLinkServiceImpl、JobLogServiceImpl、OperationLogServiceImpl、PhotoServiceImpl、UserAuthServiceImpl、UserInfoServiceImpl | 7 |
- **保留 6 处 `.intValue()`**：均来自 redis 浏览量统计或 PageUtil 分页参数转换，不属于 selectCount 链路，未动。

- **`selectCount` 返回 `Long` 后直接用于 `if (count == 0)` / `if (count > 0)` 会自动拆箱**：MyBatis-Plus 文档明确 `selectCount` 不会返回 null，但若未来 mapper 自定义 count SQL 返回 null，会触发 NPE，建议后续在 Part 2.x 加 `Optional.ofNullable(count).orElse(0L)` 防御。
- **`application-prod.yml` 现所有 prod 默认值为空字符串**：部署时必须通过环境变量提供，否则启动会失败。
- **dev 环境 JWT secret 为弱密钥**：`aurora-dev-jwt-secret-key-change-in-production`，Part 2.6 安全加固待处理。

## 9. 架构决策记录（本轮新增）

### ADR-01: authorizeHttpRequests 迁移方案
- **决策**：完整迁移，新建 `DynamicAuthorizationManagerImpl` 合并 `MetadataSource + AccessDecisionManager` 逻辑。
- **理由**：`authorizeHttpRequests` 内部用 `AuthorizationFilter`，不再使用 `FilterSecurityInterceptor`，`withObjectPostProcessor(FilterSecurityInterceptor.class, ...)` 不生效。单纯字面替换会导致动态权限校验完全失效。
- **代价**：新增 1 个类 + 修改 3 个类。

### ADR-02: FilterInvocation null 规避
- **决策**：在 `FilterInvocationSecurityMetadataSourceImpl` 中提取 `getAttributes(HttpServletRequest)` 方法，`DynamicAuthorizationManagerImpl` 直接传 request，不再构造 `FilterInvocation`。
- **理由**：Spring Security 6.x 在 `FilterInvocation` 构造函数加了 `Assert.isTrue` 检查，不允许 response/chain 为 null。
- **代价**：`FilterInvocationSecurityMetadataSourceImpl` 同时保留两个 `getAttributes` 重载（一个接收 `Object`，一个接收 `HttpServletRequest`），前者保留给原 `FilterSecurityInterceptor` 调用路径（虽然已不再使用），后者供新 `DynamicAuthorizationManagerImpl` 使用。

### ADR-03: Redis 配置前缀修正
- **决策**：`spring.redis.*` → `spring.data.redis.*`。
- **理由**：Spring Boot 3.x 把 Redis 配置前缀从 `spring.redis.*` 改为 `spring.data.redis.*`，原配置不生效导致密码丢失，引发 `NOAUTH HELLO must be called with the client already authenticated` 错误。
- **代价**：无，纯配置修正。

### ADR-04: CORS allowedOriginPatterns
- **决策**：`allowedOrigins("*")` → `allowedOriginPatterns("*")`。
- **理由**：Spring 6.x 不允许 `allowCredentials=true` + `allowedOrigins="*"` 同时存在，会抛 `IllegalArgumentException`。
- **代价**：无，纯配置修正。

### ADR-05: selectCount Long 重构范围
- **决策**：把所有 `selectCount` 调用链相关 DTO/VO/Service/Mapper/XML 的 `Integer` 改为 `Long`，移除 `.intValue()`。
- **理由**：MyBatis-Plus 3.5.9 `selectCount` 返回 `Long`，原代码用 `.intValue()` 转 `Integer`，超过 `Integer.MAX_VALUE` 会溢出。
- **代价**：影响 4 个 DTO + 10 个 ServiceImpl + 4 个 Mapper + 4 个 XML + 7 个下游 Service。
- **保留**：6 处 `.intValue()` 来自 redis 浏览量统计或 PageUtil 分页参数转换，不属于 selectCount 链路，未动。

### ADR-06: 统一返回结构 + 全局异常处理取代 /bizException
- **决策**：删除 `BizExceptionController`（通过 `/bizException` 路径抛异常的反常设计），新建 `ControllerAdviceHandler`（`@RestControllerAdvice`），覆盖 9 类异常统一返回 `ResultVO<?>`。
- **理由**：
  - 旧设计的缺陷：通过 Servlet forward `/bizException` 路径来抛异常，调用链不透明，且代码量极少（只是一个 `throw` 转发层，完全可以由 `@ExceptionHandler` 替代）。
  - 验证确认：全局搜索确认无任何代码设置 `request.setAttribute("bizException", ...)` 或 `request.getRequestDispatcher("/bizException").forward(...)`，属于死代码。
- **Risk**：`spring.web.resources.add-mappings: false` 会破坏 Knife4j 的 `/doc.html` 和 `/webjars/**` 静态资源映射，因此不加该配置，而是保留 `NoResourceFoundException` 处理器捕获真正不存在的静态资源 404。

### ADR-07: CompletableFuture.get() → .join() 替换 @SneakyThrows
- **决策**：Service 层所有 `CompletableFuture.get()` 改为 `.join()`，从而消除 `InterruptedException` / `ExecutionException` checked exception，不再需要 `@SneakyThrows`。
- **理由**：
  - `.join()` 不抛 checked exception，异常时抛 `CompletionException`（RuntimeException）。
  - 差异分析：`.get()` 抛 `ExecutionException`，`.join()` 抛 `CompletionException`，二者都会被全局 `@ExceptionHandler(Exception.class)` 捕获，返回 `SYSTEM_ERROR(50000)`，**对外接口格式完全一致**。
  - 无上层代码针对性地 try-catch `ExecutionException`，所以替换安全。
- **代价**：排查时看到的异常类型从 `ExecutionException` 变为 `CompletionException`。

### ADR-08: 日志框架统一为 @Slf4j
- **决策**：项目中所有日志注解统一使用 `@Slf4j`，不再混用 `@Log4j2`。
- **理由**：Spring Boot 3.x 底层日志框架为 Log4j2，`@Slf4j` 和 `@Log4j2` 在运行时行为等价。统一为 `@Slf4j` 避免混用，与重构方案文字一致。
- **代价**：无运行时差异，纯风格统一。

# Aurora 系统架构总览图\n\n> 本文档描述 Aurora 博客系统的整体架构、模块划分与核心调用链路，适用于重构开发与后期维护。\n\n---\n\n## 一、系统架构概览\n\n`mermaid\ngraph TB\n    subgraph 前台[前台 aurora-blog]\n        BLOG[Vue 3 + Element Plus]\n    end\n    subgraph 后台[后台 aurora-admin]\n        ADMIN[Vue 2 + Element UI]\n    end\n    subgraph 网关层[网关层]\n        Nginx[Nginx 反向代理/静态资源]\n    end\n    subgraph 后端[后端 aurora-springboot]\n        CTRL[Controller 层]\n        SVC[Service 层]\n        MAP[Mapper 层]\n        CONFIG[Security/Config]\n        MQ[RabbitMQ Consumer]\n        QUARTZ[Quartz 定时任务]\n    end\n    subgraph 数据层[数据层]\n        DB[(MySQL 9.7.1)]\n        REDIS[(Redis 7.x)]\n        ES[(Elasticsearch 8.15.x + IK)]\n    end\n    subgraph 消息[消息中间件]\n        RMQ[RabbitMQ 3.13.x]\n    end\n    subgraph 存储[对象存储]\n        OSS[阿里云 OSS / MinIO]\n    end\n    subgraph 外部[外部服务]\n        QQ[QQ 互联登录]\n        SMTP[SMTP 邮件服务]\n    end\n    BLOG --> Nginx\n    ADMIN --> Nginx\n    Nginx --> CTRL\n    CTRL --> SVC\n    SVC --> MAP\n    MAP --> DB\n    SVC --> REDIS\n    SVC --> ES\n    SVC --> OSS\n    MQ --> SVC\n    MQ --> RMQ\n    RMQ --> MQ\n    DB --> MAXWELL[Maxwell binlog 监听]\n    MAXWELL --> RMQ\n    CTRL --> CONFIG\n    QUARTZ --> SVC\n    SVC --> SMTP\n    QQ --> CTRL\n`\n\n---\n\n## 二、目录结构说明\n\n### 2.1 后端（aurora-springboot）\n\n`\ncom.aurora\n├── annotation/          # 自定义注解（@OptLog, @AccessLimit）\n├── aspect/ 文章发布流程（全链路）\n\n`mermaid\nsequenceDiagram\n    participant Admin as 后台管理员\n    participant FE as 后台前端\n    participant API as ArticleController\n    participant SVC as ArticleService\n    participant DB as MySQL\n    participant ES as Elasticsearch\n    participant RMQ as RabbitMQ\n    participant MAX as Maxwell\n    Admin->>FE: 编写文章 → 点击发布\n    FE->>API: POST /admin/articles\n    API->>SVC: saveOrUpdateArticle\n    SVC->>DB: 保存/更新 t_article\n    SVC-->>API: 返回成功\n    API-->>FE: 200 OK\n    DB->>MAX: binlog 变更\n    MAX->>RMQ: 发送变更消息\n    RMQ->>MaxWellConsumer: 消费\n    MaxWellConsumer->>ES: 更新索引\n`\n\n### 3.2 用户登录认证流程\n\n`mermaid\nsequenceDiagram\n    participant User as 用户\n    participant FE as 前端\n    participant API as UserAuthController\n    participant SVC as UserAuthService\n    participant Token as TokenService\n    participant Redis as Redis\n    User->>FE: 输入用户名密码\n    FE->>API: POST /users/login\n    API->>SVC: 认证用户\n    SVC->>Token: createToken\n    Token->>Redis: hSet(login_user, userId, ...)\n    Token-->>API: JWT\n    API-->>FE: UserInfoDTO + Token\n`\n\n---\n\n## 四、模块职责矩阵\n\n| 模块 | Controller | 核心 Service | 数据库表

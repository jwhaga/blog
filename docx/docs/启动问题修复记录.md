## 二、修复的问题

### 2.1 管理后台黑屏

**现象**：访问 localhost:8083 显示完全黑屏。

**原因**：Vue 编译进程冲突。第一次启动的 Start-Job 后台进程未正确终止，后又启动了新的 
pm run serve 实例，两个 Webpack 编译进程（各占 ~730MB）同时运行导致内存争抢，chunk-vendors.js（21MB）迟迟编译不完，浏览器请求超时，Vue 无法初始化。

**解决**：清理残留的 node 进程，只保留一套 dev server 正常编译。

---

### 2.2 博客前台黑屏（代理配置错误）

**现象**：访问 localhost:8084（博客）页面全黑，控制台数百个错误。

**原因**：博客的 ue.config.js 中 API 代理指向已不可用的演示站 https://www.linhaojun.top，而管理后台正确指向了 http://localhost:8081。

`js
// 错误配置（改前）
proxy: {
  '/api': {
    target: 'https://www.linhaojun.top',   // 站点已不可用
  }
}

// 正确配置（改后）
proxy: {
  '/api': {
    target: 'http://localhost:8081',        // 本地后端
  }
}
`

**解决**：修改 urora-vue/aurora-blog/vue.config.js 第 9 行，将 target 改为 http://localhost:8081。

---

### 2.3 博客前台黑屏（Controls.vue null 崩溃）

**现象**：代理修复后页面依然全黑，控制台大量：

`
Uncaught (in promise) TypeError: Cannot read properties of null (reading 'avatar')
    at eval (Controls.vue:114:1)
`

**原因**：Controls.vue 模板用 userInfo !== '' 判断用户是否登录，但 userStore 的初始值是 
ull 而非 ''。
ull !== '' 结果为 	rue，于是 Vue 尝试渲染 <img :src="null.avatar" />，直接崩溃。崩溃导致 Vue 组件渲染失败，页面只剩深色背景。

`html
<!-- 错误（改前）：null !== '' 为 true，进入头像分支 -->
<template v-if="userInfo !== ''">
  <img :src="userInfo.avatar" />  <!-- 崩溃 -->

<!-- 正确（改后） -->
<template v-if="!!userInfo">
  <img :src="userInfo?.avatar" />
`

**解决**：修改 src/components/Header/src/Controls.vue，将 userInfo === '' 改为 !userInfo，userInfo !== '' 改为 !!userInfo。

---

### 2.4 残余非空断言（!.）替换

**现象**：其他组件中存在类似的 !. 非空断言，在 null 时同样会引发崩溃。

**涉及文件及改动**：

| 文件 | 改动 |
|------|------|
| components/Header/src/Controls.vue | userInfo === '' → !userInfo，!== '' → !!userInfo |
| components/ArticleCard/src/HorizontalArticle.vue | 4 处 	opArticle!. → 	opArticle?. |
| components/Comment/src/CommentForm.vue | userInfo!.avatar → userInfo?.avatar |
| components/Comment/src/CommentReplyForm.vue | userInfo!.avatar → userInfo?.avatar |
| components/UserCenter.vue | 6 处读取 ?. 替换，2 处赋值加 if 守卫 |

**替换规则**：
- 读取属性：obj!.prop → obj?.prop
- 赋值操作：obj!.prop = val → if (obj) obj.prop = val

## 四、注意事项

1. **前端修改后需重启 dev server**：修改 ue.config.js 或组件源码后，需 Ctrl+C 停掉 
pm run serve 再重新启动。
2. **多实例冲突**：如果启动前端后页面一直不加载，检查是否有多个 node 进程占用同一端口，清理后重启。
3. **CDN 资源**：项目引用了 static.linhaojun.top 的静态资源（头像、Logo），该域名目前已不可用，页面中的图片会显示为占位或空白，但不影响功能。
4. **静态资源 CDN**：页面还引用了 cdnjs.cloudflare.com 的 KaTeX 样式和 connect.qq.com 的 QQ 登录 SDK，如果网络环境受限可能影响样式或功能。

## 一、前台博客（aurora-blog）

### 1.1 ESLint 警告（337 条）

```
0 errors, 337 warnings
```

| 规则 | 数量 | 说明 |
|------|------|------|
| `@typescript-eslint/no-explicit-any` | ~130 | 组件模板、工具函数中未显式类型的 `any` 参数和返回值 |
| `vue/attributes-order` | ~207 | 模板属性顺序（`v-if` 应先于 `class`，`class` 应先于 `@click` 等） |

**影响**：不阻塞构建。`no-explicit-any` 的消除依赖渐进式类型化推进，`attributes-order` 可用 `eslint --fix` 自动修复（175 条可自动修）。

**建议处理方式**：
```powershell
cd aurora-vue/aurora-blog
npm run lint:fix    # 自动修复可修复项
```
其余 `no-explicit-any` 需在后续重构中逐步替换为具体类型。

### 1.2 Sass 弃用警告

```text
@import rules are deprecated and will be removed in Dart Sass 3.0.0
Legacy JS API will be removed in Dart Sass 2.0.0
```

**根因**：项目大量使用 Sass `@import` 语法（已弃用，建议替换为 `@use`/`@forward`）。同时也使用了 Dart Sass 的旧版 JS API（`@vue/cli-service` 的 sass-loader 配置未更新）

**涉及文件**：
- `src/styles/index.scss`（多次 `@import`）
- 各 `.vue` 组件中的 `<style lang="scss">` 块

**影响**：当前版本不影响构建，未来 Dart Sass 3.0 将不再支持 `@import`。

### 1.3 组件模板中残留的 `as any`

Part 3.2 只覆盖了：
- `src/api/api.ts` 的参数类型（22 个方法）
- `src/types/blog.ts` 的类型定义（27 个接口）
- Stores 的 state 类型（8 个文件，消除 `as any`）

**未覆盖**：
- Vue 组件模板中的 `as any` 类型断言
- 组件 `setup()` 函数中的 `any` 变量声明
- 工具函数和插件的类型

**影响**：类型不安全（运行时错误更难以在编译期发现），但功能正常运行。

### 1.4 TypeScript strictNullChecks 的非空断言

在修复 65 个 TS 错误时，为快速修复使用了 `!` 非空断言：

```typescript
appStore.websiteConfig!.websiteTitle   // 代替可选链 ?.
userStore.userInfo!.avatar             // 同上
```

这些断言在运行时安全的（`websiteConfig` 在 `App.vue` onMounted 中初始化，早于子组件渲染），但风格上不如可选链 `?.` 优雅。

**涉及范围**：`App.vue`, `Avatar.vue`, `CommentForm.vue`, `CommentReplyForm.vue`, `Notice.vue`, `WebsiteInfo.vue`, `UserCenter.vue`, `HorizontalArticle.vue`

## 二、后台管理面板（aurora-admin）

### 2.1 PostCSS / Tailwind CSS 构建（同前台问题）

后台是 Vue 2 + Element UI + Vue CLI 项目，在 Node 24 下大概率会遇到和前台相同的 PostCSS/Tailwind CSS 构建崩溃。修复方法同 Part 5.1 中前台的处理方式：

1. 安装 `sass`（dart-sass）作为 devDependency
2. 升级 `tailwindcss` 到 v3（或 PostCSS 8 兼容版本）
3. 升级 `postcss` 和 `autoprefixer`

### 2.2 缺少 ESLint / Prettier 配置

后台项目没有 `.eslintrc.js` 和 `.prettierrc`，代码风格无统一约束。建议在迁移到 Vue 3 时（Part 3.6）一次性补充。

### 2.3 纯 JavaScript（非 TypeScript）

后台项目使用纯 JS 而非 TypeScript，与前台（Vue 3 + TS）的技术栈不一致。如果计划长期维护，建议迁移到 TypeScript。
